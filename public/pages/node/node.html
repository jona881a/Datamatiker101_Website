What is NodeJS?

Well, first of all Nodejs is not a language! it is a intepreter of JavaScript. But how does it intepret JavaScript when the browser usually are the one that can understand JavaScript? Well, simply by extracting the chromium V8 engine from the browser and wrapping it so Node creates a runtime environment that we can run JavaScript in

What is a REPL?

REPL stands for Read, eval, print and loop. Every programming langauge has a REPL and the purpose of the REPL are to quickly run code statements (single liners only) by refering to the languages REPL. In Node, the repl can be opened by entering node in the terminal and hitting return.

You now have an environment which can run simple JavaScript statements for testing purposes

Where do we keep dependencies in Node?

Like any other programming langauge or environment we need packages to make the program functional and to use Node to its full potential, we of course need to utilize the reason we want JavaScript code in a backend environment.

In order to do so, we first and foremost need to define the package.json file and initialize it as json, by adding {} to the document. secondly, we install a package called express by running the npm command to install it here. This creates a dependency in the package.json file with the latest stable version, which we now can import to our app.js document.

How do we then import and use libraries in NodeJS?

So now that we have installed express as a Node Module, how do we utilize it? This can be done by stating the require keyword. It looks like this:

const express = require("express");
Then we need to use what we just imported: const app = express();

What are npm?

Node package manager is the equivalent to Java's Maven build tool. Npm is also a build tool, but for NodeJS, almost every programming langauge has a build tool today, the langauges which are backend focused that is.

As discussed in earlier section, in order to use npm and install dependencies we need a file called package.json

The Role of package.json

Firstly, we need to initialize the package.json file as a json. This is done by stating {} in the file when it is created

We can then run commands to install packages, we can run npm init which sets up your project to be a npm package lik if you were to create a application to be used by others

The second thing the package.json can be utilized to do is to keep build steps (as they are called in node) by stating scripts

Scripts

In the package.json file we can specifiy scripts that can be run by the npm. Some common scripts are when we need to run our project in a development env and prodcution env. This is useful when we deploy to Heroku because Heroku needs to use npm to start app.js and if we do not specify it in the scripts part of package.json it won't be able to run the application
